HOW-TO-1: Developing inside of containers

This is a step-by-step guide on setting up containerized dev environment: VS Code + Django + React + PostgreSQL

  • Why in-container IDE?
    • If it works locally, it works in production
    • No need to rebuild the container every time you make a code change
    • Edit, debug, and test in the container
  • Dependencies
    1. Docker desktop
    1. VS Code

Steps

  1. Install VS Code extensions
    1. Remote-Container

      This is a extension developed by Microsoft team. It enables a fully featured "just like local" experience develop in Container, WSL, and SSH workspaces.

    1. Docker
  1. Create a dev container
    1. Create a working directory
      mkdir myAppContainer
cd myAppContainer
    1. Start VS Code from the terminal
      code .  ;; `code` is the VS Code command, opens VS Code IDE in current directory
    1. Create a new container using Remote-Container command

      VS Code shortcut: command + shift+ p to open the command bar, type in the following command:

      Remote-Container: Add Development Container Configuration Files...

      After this command, it will show a bunch of Microsoft pre-defined containers.

      Choose Python 3 & PostgreSQL

      Then, you will be notified by this:

      Click on Reopen in Container , then, VS code is running in the container. This left-bottom green part tells that you are developing in a dev container with Python 3 and PostgreSQL.

  1. Setup Django app & PostgreSQL
    1. Open the terminal in VS Code

      Shortcut: control^ ~

    1. Install requirements

      Create a file requirements.txt , indicates whatever you need for your project. A simple example would be

      Django
djangorestframework
psycopg2

      In the terminal, run this command to install dependencies

      pip install -r requirements.txt
    1. Create Django project
      django-admin startproject mysite
    1. Validate installation
      python manage.py runserver 0.0.0.0:8000
    1. Create Django app
      django-admin startapp newApp
    1. Migrate to PostgreSQL container

      Django has default apps in settings.py , these are pre-defined Django models.

      INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
]

      In the terminal, run this command to migrate Django models into PostgreSQL DB without directly operating on PostgreSQL

      python manage.py migrate

      After this command, click on SQLtools sign in the sidebar. Then you will see the following things. Some tables have been created from the previous migratio

  1. Setup React
    1. Install Node

      Normally, Node has been installed when you created the dev container. In the Dockerfile under .devcontainer directory, the argument INSTALL_NODE is set as true as default.

      # [Option] Install Node.js
ARG INSTALL_NODE="true"
ARG NODE_VERSION="lts/*"
RUN if [ "${INSTALL_NODE}" = "true" ]; then su vscode -c "source /usr/local/share/nvm/nvm.sh && nvm install ${NODE_VERSION} 2>&1"; fi
    1. Create a React app

      From workspace root, run this command to create a react app handily

      npx create-react-app frontend

      More details about create-react-app on this link

    1. Start your React app
      cd frontend
npm start ;; start a local server
